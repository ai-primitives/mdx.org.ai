# Component Type

## Overview

The Component type represents a reusable UI element in MDX that can combine Markdown content with JSX functionality. Components can range from simple styled elements to complex interactive features with state management and lifecycle methods.

## Properties

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `name` | `string` | Yes | The name of the component |
| `props` | `object` | No | Component properties and their types |
| `children` | `ReactNode` | No | Child elements or content |
| `style` | `object` | No | Component styling |
| `className` | `string` | No | CSS class names |
| `events` | `object` | No | Event handlers |
| `state` | `object` | No | Internal component state |
| `context` | `object` | No | Context dependencies |

## Examples

### Basic Markdown Component

```mdx
---
"@context": https://mdx.org.ai
"@type": Component
name: InfoBox
props:
  type: string
  title: string
---

export const InfoBox = ({ type = 'info', title, children }) => (
  <div className={`info-box ${type}`}>
    <h3>{title}</h3>
    <div className="content">
      {children}
    </div>
  </div>
)

<InfoBox type="warning" title="Important Note">
  This is a **markdown-enabled** component with _styling_.
</InfoBox>
```

### Interactive Component

```mdx
---
"@context": https://mdx.org.ai
"@type": Component
name: CodePlayground
props:
  language: string
  initialCode: string
---

export const CodePlayground = ({ language, initialCode }) => {
  const [code, setCode] = useState(initialCode)
  const [output, setOutput] = useState('')

  return (
    <div className="playground">
      <Editor
        value={code}
        onChange={setCode}
        language={language}
      />
      <Preview code={code} />
      <Console output={output} />
    </div>
  )
}

<CodePlayground
  language="javascript"
  initialCode="console.log('Hello MDX!')"
/>
```
